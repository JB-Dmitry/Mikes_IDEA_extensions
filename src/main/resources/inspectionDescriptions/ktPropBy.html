Reports cases of property delegation. Property delegation slows down initialization and should be avoided in applications which should start up fast.

The cause of this slowdown it that all delegation operator functions accept KProperty instances 

operator fun getValue(thisRef: R, property: KProperty<*>): T
operator fun setValue(thisRef: R, property: KProperty<*>, value: T)
operator fun provideDelegate(thisRef: R, prop: KProperty<*>): Delegate

There KProperty instances are created via reflection.
